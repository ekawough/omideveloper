About

The Developer

Omi Verified

Developer

I build apps for the Omi wearable — a device that represents a genuinely different kind of computing: ambient, voice-first, and always available. My goal is to make Omi useful in practical, everyday situations.

Tesla Voice Control was born from a simple frustration: too many taps to do simple things in a car. Omi makes it frictionless. More apps are in the works, all focused on the same principle — reduce friction, stay in the moment.

If you have ideas for integrations, feedback on existing apps, or just want to connect, reach out.

Sections

01. Data Collection 02. How It's Used 03. Third Parties 04. Your Controls

01. Information We Collect

Tesla Voice Control collects only what is necessary to execute your voice commands:

  • Tesla API Credentials

    Used in-session to authenticate with Tesla's API. Never persisted on any server.

  • Voice Commands

    Processed by Omi's on-device model. Not recorded or stored by this app.

02. How It's Used

Data is used solely to fulfill your command — the voice instruction triggers an API call, and that's it. There is no secondary use.

Command Execution

Your voice instruction is matched to a Tesla API action and fired immediately.

No Profiling

No usage history is built, no behavioral profiles are created, no patterns are analyzed.

03. Third-Party Services

The only third-party service involved is Tesla's official API:

Vehicle Control Tesla API (official)
Analytics / Tracking None
